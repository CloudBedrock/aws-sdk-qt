6 Libqtaws is an open-source (LGPL) library for consuming [Amazon Web Services (AWS)](http://aws.amazon.com/) from
7 [Qt](http://qt-project.org/) applications.
8 
9 The library is in very early stages of development; only the AWS signature
10 code and AWS endpoints information has been implemented so far, but what has been implemented is
11 [100% documented](http://pcolby.github.io/libqtaws/0.1.0/api/annotated.html)
12 (including the [internal code](http://pcolby.github.io/libqtaws/0.1.0/internal/annotated.html)) and
13 currently has 100% [unit test line coverage](http://pcolby.github.io/libqtaws/0.1.0/coverage/).
14 
15 Specific AWS services are being implemented in the order in which they are of interest to me,
16 but of course, anyone can contribute implementations for any services that are more important to them.
17 Also, feel free to use the issue tracker to show interest in particular AWS services being implemented.
18 
19 Next steps are roughly:
20 
21 1. add Amazon's official V4 signature test suite to the unit tests ([_done_](https://github.com/pcolby/libqtaws/issues/2))
22 2. implement base AwsService class(es) (_in progress_)
23 3. SQS (_in progress_)
24 4. SNS
25 5. DynamoDB
26 6. S3, EC2, Elasticache, ..., etc.

28 ## Requirements
29 ### Qt 5.1
30 Currently, libqtaws requires Qt 5.1+ This is because the library uses Qt's QMessageAuthenticationCode class for
31 generating HMAC codes for AWS request signatures, and that class was not added until Qt 5.1.0.
32 
33 It is possible to make the library work with Qt 5.0 by backporting QMessageAuthenticationCode from Qt 5.1 to Qt 5.0 -
34 indeed, this is what this project does for [travis-ci](https://travis-ci.org/) support (see [.travis.yml](.travis.yml)
35 for example), but that is not officially supported (might become so, if enough people show interest).
36 
37 Note, the library will soon depend on Qt5's built-in JSON support, so it is unlikely that libqtaws will ever officially
38 support earlier version of Qt, such as Qt4.8.

40 ## Building
41 This library supports out-of-source builds (highly recommended), for example:
42 
43 ~~~{.sh}
44 mkdir -p <tmp-build-dir>
45 pushd <tmp-build-dir>
46 qmake -Wall -Wlogic -Wparser CONFIG+=debug_and_release <path-to-cloned-repo>
47 make # or `nmake` on Windows
48 make check
49 popd
50 ~~~
