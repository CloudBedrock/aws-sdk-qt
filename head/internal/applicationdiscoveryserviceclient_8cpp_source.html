31 /**
32  * @class ApplicationDiscoveryServiceClient
33  *
34  * @brief Client for AWS Application Discovery Service
35  *
36  * <fullname>AWS Application Discovery Service</fullname>
37  *
38  * AWS Application Discovery Service helps you plan application migration projects by automatically identifying servers,
39  * virtual machines (VMs), software, and software dependencies running in your on-premises data centers. Application
40  * Discovery Service also collects application performance data, which can help you assess the outcome of your migration.
41  * The data collected by Application Discovery Service is securely retained in an AWS-hosted and managed database in the
42  * cloud. You can export the data as a CSV or XML file into your preferred visualization tool or cloud-migration solution
43  * to plan your migration. For more information, see <a href="http://aws.amazon.com/application-discovery/faqs/">AWS
44  * Application Discovery Service
45  *
46  * FAQ</a>>
47  *
48  * Application Discovery Service offers two modes of
49  *
50  * operation> <ul> <li>
51  *
52  * <b>Agentless discovery</b> mode is recommended for environments that use VMware vCenter Server. This mode doesn't
53  * require you to install an agent on each host. Agentless discovery gathers server information regardless of the operating
54  * systems, which minimizes the time required for initial on-premises infrastructure assessment. Agentless discovery
55  * doesn't collect information about software and software dependencies. It also doesn't work in non-VMware environments.
56  *
57  * </p </li> <li>
58  *
59  * <b>Agent-based discovery</b> mode collects a richer set of data than agentless discovery by using the AWS Application
60  * Discovery Agent, which you install on one or more hosts in your data center. The agent captures infrastructure and
61  * application information, including an inventory of installed software applications, system and process performance,
62  * resource utilization, and network dependencies between workloads. The information collected by agents is secured at rest
63  * and in transit to the Application Discovery Service database in the cloud.
64  *
65  * </p </li> </ul>
66  *
67  * We recommend that you use agent-based discovery for non-VMware environments and to collect information about software
68  * and software dependencies. You can also run agent-based and agentless discovery simultaneously. Use agentless discovery
69  * to quickly complete the initial infrastructure assessment and then install agents on select
70  *
71  * hosts>
72  *
73  * Application Discovery Service integrates with application discovery solutions from AWS Partner Network (APN) partners.
74  * Third-party application discovery tools can query Application Discovery Service and write to the Application Discovery
75  * Service database using a public API. You can then import the data into either a visualization tool or cloud-migration
76  *
77  * solution> <important>
78  *
79  * Application Discovery Service doesn't gather sensitive information. All data is handled according to the <a
80  * href="http://aws.amazon.com/privacy/">AWS Privacy Policy</a>. You can operate Application Discovery Service offline to
81  * inspect collected data before it is shared with the
82  *
83  * service> </important>
84  *
85  * Your AWS account must be granted access to Application Discovery Service, a process called <i>whitelisting</i>. This is
86  * true for AWS partners and customers alike. To request access, <a
87  * href="http://aws.amazon.com/application-discovery/">sign up for Application Discovery Service</a>.
88  *
89  * </p
90  *
91  * This API reference provides descriptions, syntax, and usage examples for each of the actions and data types for
92  * Application Discovery Service. The topic for each action shows the API request parameters and the response.
93  * Alternatively, you can use one of the AWS SDKs to access an API that is tailored to the programming language or platform
94  * that you're using. For more information, see <a href="http://aws.amazon.com/tools/#SDKs">AWS
95  *
96  * SDKs</a>>
97  *
98  * This guide is intended for use with the <a href="http://docs.aws.amazon.com/application-discovery/latest/userguide/">
99  * <i>AWS Application Discovery Service User Guide</i>
100  */
