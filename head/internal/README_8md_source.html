10 Libqtaws is an open-source (LGPL) library for consuming [Amazon Web Services (AWS)](http://aws.amazon.com/) from
11 [Qt](http://qt-project.org/) applications.
12 
13 The library is still in early stages of development, but already includes much
14 of the basic framework plumbing required for implementing asynchronous AWS
15 clients, including generic classes for managing the building of AWS requests
16 (including all five AWS signature types) and processing of AWS responses.
17 
18 The first [0.1.0](https://github.com/pcolby/libqtaws/releases/tag/v0.1.0)
19 release included just the signature, endpoint and region classes. But that
20 release was very [100% documented](http://pcolby.github.io/libqtaws/0.1.0/api/annotated.html)
21 (including the [internal code](http://pcolby.github.io/libqtaws/0.1.0/internal/annotated.html))
22 and 100% [unit tested](http://pcolby.github.io/libqtaws/0.1.0/coverage/).
23 
24 Work is now progressing on the SQS client classes - the basics of which are in
25 place now, but need to be documented and unit tested before I could consider
26 them complete.
27 
28 After SQS, other Amazon services (such as SNS, DynamoDN, S3, EC2, etc) will be
29 added in the order people ask for them.
30 
31 Of course, if you want to contribute your own classes for access specific AWS
32 services, please do so! See the `SqsClient` and related classes for the
33 patterns to use when adding new AWS service client classes.
34 
35 ## Requirements
36 ### Qt 5.1
37 Libqtaws requires Qt 5.1+ The library uses Qt's QMessageAuthenticationCode
38 class for generating HMAC codes for AWS request signatures, and that class was
39 first added in Qt 5.1.0.
40 
41 It is possible to make the library work with Qt 5.0 by backporting
42 QMessageAuthenticationCode from Qt 5.1 to Qt 5.0 - indeed, this is what this
43 project does for [travis-ci](https://travis-ci.org/) support
44 (see [.travis.yml](.travis.yml) for example), but that is not officially
45 supported (might become so, if enough people show interest).
46 
47 Note, the library will soon depend on Qt5's built-in JSON support too, so it is
48 unlikely that libqtaws will ever officially support version of Qt earlier than
49 5.0.

51 ## Building
52 This library supports out-of-source builds (highly recommended), for example:
53 
54 ~~~{.sh}
55 mkdir -p <tmp-build-dir>
56 pushd <tmp-build-dir>
57 qmake -Wall -Wlogic -Wparser CONFIG+=debug_and_release <path-to-cloned-repo>
58 make # or `nmake` on Windows
59 make check
60 popd
61 ~~~
