void SqsAddPermissionRequest::setPermission ( const QString &  accountId,
const PermissibleAction  action,
const bool  permitted = true 
)

Set a permissions to be added for a given principal.

Note, this adds or removes the action permission for accountId, within the context of this request only. That is, if the principal accountId already has the action permission granted on the remote SQS service, this request will not be modifying that existing permission; see SqsRemovePermissionRequest for that.

Parameters
accountIdPrincipal to set permssions for.
actionAction to permit or not.
permittedWhether action should be permitted or not for accountId.
void SqsAddPermissionRequest::setPermissions ( const QString &  accountId,
const PermissibleActions &  actions 
)

Set the permissions to be added for a given principal.

Note, this sets the permissions for accountId, replacing any existing permissions - but only within the context of this request. That is, if the principal accountId has other permissions alredy set on the remote SQS service, this request will have no affect on those (or rather will be in addition to those).
