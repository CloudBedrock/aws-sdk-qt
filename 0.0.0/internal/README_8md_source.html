5 Libqtaws is an open-source (LGPL) library for consuming [Amazon Web Services (AWS)](http://aws.amazon.com/) from
6 [Qt](http://qt-project.org/) applications.
7 
8 The library is in very early stages of development; only the AWS signature
9 code and AWS endpoints information has been implemented so far, but what has been implemented is
10 [100% documented](http://pcolby.github.io/libqtaws/doc/api/annotated.html)
11 (including the [internal code](http://pcolby.github.io/libqtaws/doc/internal/annotated.html)) and
12 currently has 100% [unit test line coverage](http://pcolby.github.io/libqtaws/test/coverage/).
13 
14 Specific AWS services are being implemented in the order in which they are of interest to me,
15 but of course, anyone can contribute implementations for any services that are more important to them.
16 Also, feel free to use the issue tracker to show interest in particular AWS services being implemented.
17 
18 Next steps are roughly:
19 
20 1. add Amazon's official V4 signature test suite to the unit tests - [**Done**](https://github.com/pcolby/libqtaws/issues/2)
21 2. implement base AwsService class(es)
22 3. SQS
23 4. SNS
24 5. DynamoDB

26 ## Requirements
27 ### Qt 5.1
28 Currently, libqtaws requires Qt 5.1+ This is because the library uses Qt's QMessageAuthenticationCode class for
29 generating HMAC codes for AWS request signatures, and that class was not added until Qt 5.1.0.
30 
31 It is possible to make the library work with Qt 5.0 by backporting QMessageAuthenticationCode from Qt 5.1 to Qt 5.0 -
32 indeed, this is what this project does for [travis-ci](https://travis-ci.org/) support (see [.travis.yml](.travis.yml)
33 for example), but that is not officially supported (might become so, if enough people show interest).
34 
35 Note, the library will soon depend on Qt5's built-in JSON support, so it is unlikely that libqtaws will ever officially
36 support earlier version of Qt, such as Qt4.8.
